  <action>
This is a human verification checkpoint. All deployment code is already pushed to GitHub. The user deploys via Render and Vercel dashboards (see user_setup), then verifies the live URLs work end-to-end.

What was built: A Dockerfile with dynamic PORT, 8 serving artifacts committed, render.yaml for Render config, vercel.json with SPA rewrite, and URL state sync for deep linking.

Deployment steps for the user:

**Step 1: Deploy backend to Render**
1. Go to https://dashboard.render.com/
2. Click "New" -> "Web Service"
3. Connect your GitHub repo (philly-vibe-map)
4. Render should auto-detect the Dockerfile. Verify settings:
   - Name: `vibe-mapper-api` (or your preference)
   - Plan: Free
   - Docker is selected as runtime
5. Click "Create Web Service"
6. Wait for the build to complete (3-5 minutes)
7. Note the URL (e.g., `https://vibe-mapper-api.onrender.com`)
8. Test: visit `https://YOUR-RENDER-URL/health` -- should return `{"status":"ok","artifacts_loaded":true}`
9. Test: visit `https://YOUR-RENDER-URL/neighbourhoods` -- should return GeoJSON

**Step 2: Deploy frontend to Vercel**
1. Go to https://vercel.com/new
2. Import your GitHub repo
3. Set Root Directory to `frontend`
4. Set Framework Preset to "Vite"
5. Add Environment Variable: `VITE_API_URL` = `https://YOUR-RENDER-URL` (the Render URL from Step 1, NO trailing slash)
6. Click "Deploy"
7. Wait for build to complete (1-2 minutes)
8. Visit the Vercel URL -- the full interactive map should load

**Step 3: Verify end-to-end**
1. The map shows coloured neighbourhood polygons
2. Click a neighbourhood -- sidebar opens with vibe data
3. Drag the time slider -- colours change
4. Check the URL bar -- it should show `?nid=XXX&year=YYYY`
5. Copy that URL, open a new tab, paste it -- the same neighbourhood and year should be selected
6. Open browser DevTools Network tab -- all API requests go to the Render URL with 200 responses (no CORS errors)
