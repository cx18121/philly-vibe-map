## Color

### Surface Colours

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a2e` | Map background, sidebar background, time slider panel background |
| Secondary (30%) | `#2a2a4a` | Time slider track background, vibe bar track, tooltip background |
| Accent (10%) | see Archetype Palette below | Neighbourhood fills, vibe bars, slider thumb, play button icon |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for: neighbourhood polygon fills, vibe bar fills, the slider thumb fill (matches dominant vibe of map centre), and the play/pause button icon colour.

### Archetype Palette (inherited from Phase 4)

| Archetype | Hex | CSS var equivalent |
|-----------|-----|--------------------|
| artsy | `#882255` | inline use only |
| foodie | `#CC6677` | inline use only |
| nightlife | `#44AA99` | inline use only |
| family | `#117733` | inline use only |
| upscale | `#DDCC77` | inline use only |
| cultural | `#332288` | inline use only |
| fallback | `#888888` | used when no vibe data exists for a year |

Source: `src/lib/colors.ts` VIBE_COLORS -- Wong-adapted colourblind-safe palette from Phase 4.

### Glow and Highlight Colours

| Element | Value | Notes |
|---------|-------|-------|
| Hover highlight overlay | `#ffffff` at 15% opacity | White fill layer on top of base fill, visible only on hover via feature-state |
| Ambient glow (CSS drop-shadow) | Archetype colour at 40% opacity, 8px blur | Applied via CSS `filter: drop-shadow(0 0 8px {color}40)` on hovered feature's parent |
| Hover glow intensified | Archetype colour at 70% opacity, 12px blur | `filter: drop-shadow(0 0 12px {color}B3)` when hovered |

### Interpolation Colours

During year transitions, neighbourhood fill colours are determined by computing the interpolated vibe vector between year N and year N+1, then selecting the dominant archetype's colour from the palette above. There is no colour blending between two archetype colours -- the fill snaps to whichever archetype is dominant in the interpolated vector. The smooth effect comes from opacity and the gradual shift in which archetype wins dominance.

## Interaction Contract

### TimeSlider Interactions

| Trigger | Response | Timing |
|---------|----------|--------|
| Drag slider thumb | Map fill colours update to reflect the selected year's vibe data | Immediate (per input event) |
| Release slider thumb | Year snaps to nearest integer; map shows that year's exact data | Immediate |
| Click play button | Animation begins: year advances from current to max at 1 year per 1.5 seconds; fill colours interpolate smoothly between years | Continuous until paused or max year reached |
| Click pause button | Animation stops at current fractional year position; year label shows nearest integer | Immediate |
| Animation reaches max year | Loops back to min year and continues playing | Automatic |
| Keyboard: left/right arrow on focused slider | Decrements/increments year by 1 | Immediate (native range input behaviour) |

### Hover Glow Interactions

| Trigger | Response | Timing |
|---------|----------|--------|
| Mouse enters neighbourhood polygon | Highlight layer opacity transitions to 15% white overlay; CSS drop-shadow glow appears at 40% archetype colour | 150ms transition |
| Mouse leaves neighbourhood polygon | Highlight layer opacity returns to 0%; glow fades out | 150ms transition |
| Neighbourhood is selected (clicked) | No additional glow change beyond existing sidebar open behaviour | N/A |

### Sidebar Stagger Animation

| Trigger | Response | Timing |
|---------|----------|--------|
| Neighbourhood selected (detail data loads) | DetailPanel children animate in sequentially: VibeBars first, then SentimentPills, then TopicList, then QuoteCarousel | 80ms stagger delay between children; each child: 300ms duration, ease-out, translate Y from 12px to 0 + opacity 0 to 1 |
| Different neighbourhood selected while sidebar is open | Content re-keys on neighbourhood_id; old content exits, new content staggers in | Exit: 150ms fade out; Enter: same stagger as above |

### Animation Timing Constants

| Constant | Value | Usage |
|----------|-------|-------|
| YEAR_ADVANCE_INTERVAL | 1500ms | Time between advancing one full year during playback |
| INTERPOLATION_FRAME_RATE | 60fps (via rAF) | Colour interpolation updates during playback |
| STAGGER_DELAY | 80ms | Delay between each sidebar child animation start |
| CHILD_ENTER_DURATION | 300ms | Duration of each sidebar child entering |
| CHILD_ENTER_EASE | easeOut | Easing curve for sidebar child entrance |
| HOVER_GLOW_TRANSITION | 150ms | Fade in/out for highlight layer and CSS glow |
| SIDEBAR_SPRING_DAMPING | 25 | Existing sidebar slide spring damping (no change) |
| SIDEBAR_SPRING_STIFFNESS | 300 | Existing sidebar slide spring stiffness (no change) |

## Accessibility Contract

| Concern | Implementation |
|---------|---------------|
| Slider keyboard control | Native `<input type="range">` provides arrow key support |
| Play/pause keyboard | Standard `<button>` element, focusable, activates on Enter/Space |
| Screen reader: current year | `aria-valuetext` on range input set to year string, e.g., "2021" |
| Screen reader: play state | `aria-label` toggles between "Play timeline" and "Pause timeline" |
| Screen reader: time controls | Container has `role="group"` with `aria-label="Time controls"` |
| Reduced motion | When `prefers-reduced-motion: reduce` is active: skip interpolation frames (snap directly to target year colours), disable sidebar stagger (render all children simultaneously), disable hover glow transition |
| Colour contrast | Year display (`#e0e0e0` on `#1a1a2e`): contrast ratio 10.5:1 (passes AAA). Endpoint labels (`#999999` on `#1a1a2e`): contrast ratio 4.6:1 (passes AA for small text at 12px). |
