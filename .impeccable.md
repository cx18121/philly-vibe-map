# Impeccable Design Context

## Design Context

### Users
General public exploring Philadelphia neighbourhoods — casual browsers curious about local character and how it changes over time. Secondary audience: technical interviewers assessing full-stack craft and product judgment. Users arrive with a question ("what's my neighbourhood like?") or pure curiosity, not a research agenda. They should be able to orient immediately and be rewarded for deeper exploration.

### Brand Personality
**Curious · Precise · Alive**

Minimal and sharp enough to feel authoritative, but warm and playful enough to invite discovery. Not a dashboard — closer to an editorial data essay that moves. The data is rich and the aesthetic should honour it without overwhelming the viewer.

### Aesthetic Direction
**References:** The Pudding / NYT Graphics (editorial polish, storytelling with data) and Spotify Wrapped / Linear (bold color confidence, opinionated dark interfaces, satisfying micro-interactions).

**Anti-references:** Generic BI dashboard (Tableau/Power BI aesthetic), cluttered GIS tool, corporate "smart city" look, anything that feels like a slide deck.

**Theme:** Dark-first. Navy `#1a1a2e` base is right. The six vibe colors (Wong-adapted) are the hero — everything else should make room for them. The map is the centrepiece; UI chrome should be minimal and recede.

**Emotional goal:** Aesthetic pleasure first. Users should find it beautiful to look at even passively. Then discovery kicks in — temporal animation reveals neighbourhood change, panel opens to reward curiosity.

### Design Principles

1. **Beauty earns attention.** Aesthetic pleasure is the primary emotional goal. Every spacing decision, animation timing, and typographic choice should be deliberate. If it looks rough, it distracts from the data.

2. **Chrome recedes, data shines.** The six vibe colors and the map are the heroes. Panel backgrounds, controls, and structural elements should be dark and unobtrusive — supporting actors, not competing with the headliner.

3. **Precision over decoration.** No gratuitous flourishes. Animations serve meaning (spring physics = panel has weight, stagger = data loads progressively). Typography is functional but considered.

4. **Warmth in the details.** Micro-copy, hover states, and transitions should feel human — a little personality and wit without breaking the precision. Think Spotify Wrapped's joy within a tight grid.

5. **Ship-quality polish.** No rough edges, no half-finished states, no placeholder copy. The standard is: a senior engineer looking at this assumes it's a real product. Loading, empty, and error states are all polished.

## Tech Stack Reference
- React 19 + TypeScript + Vite
- MapLibre GL 5 + react-map-gl 8
- Framer Motion 12 (spring animations, stagger, reduced-motion support)
- Zustand 5 (map selection, temporal playback state)
- System font stack (no custom fonts currently)
- Inline styles throughout (no CSS modules or Tailwind)
- Dark basemap: CartoDB Dark Matter GL
